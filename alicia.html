專案亮點

0→1 獨立產品驗證

從實體工作坊洞察市場痛點，獨立完成競品分析、Prompt 策略至 GAS 全端實作，成功將產品部署於 LINE 生態圈，驗證 MVP 市場契合度。

系統韌性與優雅降級策略

針對 LLM 常見的 API 限流（429）與伺服器不穩定（503）挑戰，設計「優雅降級」方案。透過 Auto-Retry 自動重試機制與「擬人化緩衝」UX 文案，將技術報錯轉化為可預期的 UX 互動，確保極限負載下的用戶信任度。

混合式介面策略 GUI + CUI

創立 LINE 圖文選單與生成式對話的動態結合，利用 GUI 視覺導航「降維」複雜的 Prompt 門檻，實證有效降低新手認知負荷並提升首波留存。

高階 Prompt 工程・動態分流

導入 XML 標籤系統實作情境分流，成功解決「指令溢出」與「後設評論」問題，精準控制 AI 邊界，達成具備沉浸感的專業教練體驗。

三階段敏捷迭代 V1 → V3

V1 對話 2–3 回中止 → V2 延伸至 20 回（遭遇 API 限流）→ V3 解決技術穩定性後，80% 用戶實現 2–4 天主動回訪習慣。

專案起源與挑戰

UX 設計的學習資源從來不缺 —— YouTube 教學、線上課程、ChatGPT 隨問隨答。
但我在籌辦線下 UIUX 工作坊的第一線觀察到一個矛盾：資源越多，轉職者反而越焦慮。

原因是被動接收知識，不等於建立設計思維。

⭐ Ver.1・初始驗證：純對話原型的瓶頸

用戶表現：平均對話 2–3 回即中止，10 人初期原型驗證獲得極具啟發性的產品修正洞察。

客群專屬痛點 · User-Specific Insights

Persona A・零基礎探索者：交互冷啟動障礙

初期測試純文字版時，發現面對高度自由的文字框，初學者因缺乏 UX 知識框架，陷入「不知從何問起」的恐懼，導致產品在第一哩路就產生極高的流失率。

Persona B・初階轉職/從業者：幻覺引發的信任危機

大模型預設的「過度禮貌」與「空泛讚美」對具備基礎理論的從業者無效。使用者因無法判斷 AI 建議是否符合真實業界標準，對 AI 的產出抱持高度懷疑。

系統體驗痛點 · Systemic UX Failures

資訊過載引發的認知疲勞 初版未設定 Token 輸出限制，AI 產生過長的「文字牆」，導致行動端需反覆滑動，喪失閱讀耐心。
移動端適配失靈與格式不穩定 缺乏結構化約束，頻繁出現複雜 Markdown 符號，增加使用者的學習負荷。
指令邊界鬆散導致的語意發散 初期 System Prompt 規範較鬆散，AI 回覆過於囉唆且缺乏重點，無法在對話初期提供明確、簡潔的行動指引，導致產品易用性低於預期。

人物誌 Persona

透過兩輪驗證（初始需求訪談 vs. 原型實測），重新定義目標使用者輪廓：

#知識焦慮 #資訊過載 #名詞混淆

【Prompt 焦慮】與【空白畫布症候群】

面對純文字輸入框，不知如何精準下指令，導致高認知負荷與低啟動率。

核心痛點

缺乏名詞框架，陷入資訊過載，需要產品化解「第一句話怎麼問」的破冰阻力。

核心需求

極低門檻的認知鷹架與具體學習路徑。

#面試焦慮 #消除幻覺 #優化現有履歷
AI 互動障礙

【Output 信任度低】與【缺乏產業對齊】

通用 LLM 大模型常給予空泛讚美或幻覺，無法針對履歷提供具備 ATS 標準的量化反饋。

核心痛點

缺乏批判性回饋，無法在邏輯推演中驗證自我決策的穩健度。

核心需求

需要一個具備業界標準、能精準糾錯並模擬高壓面試的「磨刀石」。

⭐ Ver.2・上線實測：對話延伸至 20+ 回合

用戶表現：平均對話輪次從 2–3 回提升至 20 回

1

CUI + GUI 混合式介面・零摩擦破冰

問題假設：面對高度發散的 LLM 輸入框，零經驗新手容易產生「空白畫布焦慮」導致放棄。若能將「生成式輸入」轉為「選擇式點擊」，能有效降低認知負荷並提升啟動活躍率。

解決方案：導入 LINE 圖文選單（GUI），設計「我是 UX 小白」等快捷按鈕。點擊後觸發後端 System Prompt，動態拋出 5 個預設的具體探索方向，將 AI 溝通降維為點擊選擇。

2

碎片化的學習場景・情境抽測

問題假設：進階 UX 用戶不需要名詞解釋，而是缺乏面試連環追問的「實戰經驗與自信」。透過碎片化的「情境抽測」，能幫助他們建立面試的肌肉記憶。

解決方案：設計「抽測一題 UX 考題」功能，封印 AI 直接給予解答的預設行為，以「蘇格拉底教學法」反向追問引導用戶獨立推演問題與解法，打造思維訓練。

3

領域知識把關機制・Few-shot Prompting

問題假設：使用者對通用型 AI 缺乏信任，主因是其容易給予無效的罐頭稱讚或偏離 ATS 篩選標準。若能收束 AI 人設並注入評估框架，能大幅提升回饋含金量。

解決方案：針對履歷健檢場景，強制 AI 切換為「嚴厲面試官」人設，導入 Few-shot Prompting，嚴格規定輸出【❌ 模糊寫法 vs ✅ ATS 友善寫法】的對照組。

4

防流失的情緒價值設計

問題假設：轉職旅程中的「冒牌者症候群」極易引發挫折流失。若產品能在高壓糾錯的同時提供情緒安全網，將能延長用戶留存生命週期。

解決方案：植入情緒感測指令，偵測到用戶迷惘低落時，暫停高壓審查，改採三明治回饋進行同理引導，確保嚴格的 AI 教練同時具備接住用戶情緒的能力。

⚠ Ver.2 新痛點與挑戰

全域規則溢出 嚴格履歷審核規則缺乏情境隔離，導致嚴厲糾錯「溢出」到新手歡迎語，在錯誤時機給予不必要壓力。
後設評論破壞沈浸感 AI 習慣「宣告行為」（如：我現在將使用蘇格拉底式方法），自言自語中斷對話流暢度，降級為程式數據交換。
技術極限引發體驗中斷 隨對話複雜度提升，系統頻繁觸發 429 限流與 503 超載，用戶被迫直面技術報錯，嚴重摧毀沈浸感與可靠度。

⭐ Ver.3・架構重構：達成週期性使用習慣

最終驗證結果：首批用戶 2–3 天後主動回訪，達成重複使用黏性。

1

導入 XML 標籤隔離・情境分流架構

重構 Prompt 架構，利用 <Dynamic_Scenarios> 與反向提示（Negative Prompts），精準封鎖人設邊界，徹底解決指令溢出，確保對話 100% 符合當下情境。

dynamic_scenarios.prompt 
// 利用 XML 自定義標籤 和 反向提示規範（部分 prompt）

<Dynamic_Scenarios>
  根據用戶當前語意與狀態，自動分流至以下情境，
  嚴禁混合情境輸出：

  [Scenario 1: 零基礎小白]
  - 觸發條件：不懂 UX 或觸發新手按鈕。
  - ⚠ 安全護欄：絕對禁止提供對照組或拋出嚴厲的實戰
    反問，將認知負載降至最低。
</Dynamic_Scenarios>
2

實施行動優先的隱性護欄・Show, don't tell

導入「演而不說」的強制約束原則，封印 AI 解釋後台邏輯的慣性，縮短無效廢話，將首則回覆時間降至最低，找回真人般俐落的沉浸式互動。

no_meta_comment.prompt 
// 處理 AI 後設評論（部分 prompt）

<Dynamic_Scenarios>
  [Scenario 2: 求職把關與多模態履歷解析]
  - 核心指令：
    封印後設評論：【絕對禁止】向用戶解釋 STAR、
    ATS 標準、對照組格式或任何系統規則。
</Dynamic_Scenarios>
3

極限情境的優雅降級・Auto-Retry 容錯機制

開發 Auto-Retry 後端自動重試容錯機制，針對無法避免的 429/503 崩潰事件，攔截並轉譯為擬人化的優雅安撫文案，用友善的 UX 文案包裹 AI 技術極限，維持體驗的連續性。

graceful_degradation.js 
// A. 處理 429 流量限制
if (status === 429) {
  let errorMsg = responseText;
  try {
    errorMsg = JSON.parse(responseText).error.message;
  } catch (e) {}

  if (errorMsg.includes("Quota") || errorMsg.includes("Daily")) {
    return "【系統提醒】Alicia 今天的體力值已耗盡。🪫\n\n"
         + "因為我的大腦是由 Google 贊助的，每天下午 4:00\n"
         + "會重新補滿體力。可以明天準時再找我喔！😊";
  } else {
    return "【系統提醒】剛才回得太快，思緒有點打結。🌀\n\n"
         + "請休息 5 分鐘再重新傳送一次，我就能正常運作了！\n\n"
         + "✨ 建議：傳送前先點選「清空記憶」，能讓我更輕快喔！";
  }
}

// B. 處理伺服器報錯 500 / 503
if (status !== 200) {
  return "💡 哎呀，剛才訊號似乎被全球開發者擠斷了（系統標註："
       + status + "）。\n\n請重新傳送剛才的內容，讓我再試一次！";
}

專案學習與反思

  • 敏捷思維驅動的 Pivot：透過原型測試及早捕捉新手的「空白畫布焦慮」，促使產品果斷從純文字對話（CUI）轉向混合介面（GUI + CUI），在極低開發成本下避開了無效功能的堆疊，實現 MVP 資源效益最大化。
  • 技術報錯也是設計機會：面對 API 限流與延遲挑戰，利用「優雅降級」思維，透過防呆機制與友善 UX 文案，將系統死角轉化為合乎預期的順暢體驗。
  • Prompt 工程是新的 UX：AI 邊界的控制就是產品體驗的邊界。當收攏對話邊界和邏輯定義越清楚，才能精準掌控 LLM 輸出穩定度，進而增進使用體驗。
  • 面對尚未驗證的假設：目前 MVP 驗證了使用黏性（80% 回訪率、平均回訪 3 次），但核心假設「蘇格拉底法是否真正提升設計思維能力」尚缺乏量化佐證。下一步將設計前後測問卷，對照用戶在使用前後分析 UX 案例的能力差異。

下一步計畫

  • RAG 架構導入：將過往優質的履歷範本向量化，導入 RAG 架構（檢索增強生成），作為 AI 回覆的真理依據（Ground Truth），以實體數據進一步抹除幻覺機率，拉升服務的高階含金量。
  • 付費可行性探索：探索「訂閱制」付費可能性，或授權給 UX 培訓機構作為輔助工具的商業模式。
  • 學習效果量化：設計前後測問卷，量化驗證「蘇格拉底法是否真正提升設計思維能力」的核心假設，為產品商業化提供數據佐證。
